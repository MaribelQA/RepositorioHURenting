# HU-136903 — CI EXPRESS OPTIMIZAR TIEMPO DE CARGA

**ID**: 136903

**Titulo**: CI EXPRESS OPTIMIZAR TIEMPO DE CARGA

**Descripción**:
Como Usuario del sistema CI y CI Express
Quiero que el formulario de carga inicial abra de manera ágil,
Para iniciar mi operación sin interrupciones ni tiempos de espera prolongados que afecten mi productividad.

## Criterios de Aceptación

El problema ocurre al ejecutar el código que inicializa el módulo (según la imagen proporcionada).
Captura de pantalla 2026-05-05 095857.png

(NO TRASLADAR AL MÓDULO DE ÓRDENES)
No se puede trasladar la responsabilidad a otro módulo, ya que solo movería el problema.
El objetivo es reducir el tiempo de carga sin afectar la funcionalidad actual.
Entorno: Aplicación (probablemente escritorio en .NET/C#).
El tiempo actual de carga es ~29 segundos, lo que afecta la experiencia del usuario.

## Criterios de Aceptación

El tiempo de carga del módulo FRMcargar inicial debe reducirse al menos en un 50% (ideal: menos de 10 segundos).
La funcionalidad del módulo debe mantenerse intacta (sin pérdida de datos ni errores).
Debe existir un log de métricas (usuario, fecha y hora) que registre el tiempo de carga antes y después de la optimización.
Las pruebas deben confirmar que la mejora se mantiene en diferentes entornos (desarrollo, QA, producción).
No se deben generar dependencias adicionales que afecten otros módulos.
Analizar el código actual del módulo FRMcargar inicial para identificar cuellos de botella.
Optimizar consultas a base de datos (uso de índices, reducción de joins innecesarios).
El formulario debe mostrar un estado de "Cargando" (spinner o barra de progreso (bombillo CI)) si el proceso supera los 2 segundos, permitiendo que la ventana sea movible o cancelable.
Agregar logs para medir tiempos de carga antes y después.
